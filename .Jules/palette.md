## 2024-05-24 - Interactive Quiz Options Custom Keyboard Accessibility
**Learning:** When using `div` elements as custom radio buttons (`.option`) for quiz questions, they naturally lack keyboard interactivity and semantic meaning for screen readers. Users navigating via keyboard (Tab) cannot access or select them without adding `tabindex="0"`, appropriate ARIA roles (`role="radiogroup"` and `role="radio"`), dynamic state tracking (`aria-checked`), and custom `keydown` listeners handling Enter/Space keys (while preventing default Space scrolling).
**Action:** Always implement the `radiogroup` and `radio` ARIA pattern along with `tabindex` and `keydown` handlers when building custom interactive option components using non-interactive HTML tags (like `div`) to ensure keyboard users and screen readers can successfully interact with the UI.
