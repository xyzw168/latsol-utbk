## 2024-05-01 - Custom Radio Group Accessibility
**Learning:** Interactive quiz options (`.option`) acting as a group require custom keyboard accessibility support. Standard `div` elements are not focusable or operable via keyboard by default, making the application inaccessible to users relying on keyboard navigation or screen readers. Providing `:focus-visible` styling is essential to ensure visual feedback without affecting mouse users.
**Action:** When implementing custom options (like an `#options-box` containing `.option` elements), always add `role="radiogroup"` to the container, and `role="radio"`, `tabindex="0"`, `aria-checked` attributes to the options. Additionally, implement keydown event listeners for `Enter` and `Space` (preventing default scrolling for Space) to toggle selection, and ensure DOM queries for option toggling are scoped to the container for performance.
