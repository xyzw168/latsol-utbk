## 2026-04-30 - Accessible Interactive Custom Radio Options
**Learning:** Custom interactive elements representing a list of choices (like quiz options) must have keyboard navigation support and appropriate ARIA roles to be fully accessible. Divs acting as radio buttons need `role="radio"`, `aria-checked`, `tabindex="0"`, and event listeners for 'Enter' and 'Space' keypresses (using `e.preventDefault()` on Space to prevent scrolling). The parent container must have `role="radiogroup"` and be labelled properly. Additionally, `:focus-visible` styles should be implemented so keyboard users can see where focus is placed without affecting mouse/touch users.
**Action:** When implementing custom selection components instead of native `<input type="radio">`, always enforce the full `role="radio"` / `role="radiogroup"` pattern along with `tabindex`, keyboard event listeners, and visible focus indicators.
