## 2024-05-14 - Keyboard Accessibility for Custom Quiz Options
**Learning:** In dynamically generated quiz options structured with generic `div` elements, adding `role="radio"` and keyboard event listeners (`keydown` for Enter and Space) is critical. Setting `tabindex="0"` allows users to tab into the options, but does not provide interactive selection via keyboard. Additionally, the `:focus-visible` pseudo-class should be implemented so keyboard navigators have a clear visual focus indicator, without relying entirely on `.selected` styling which may not trigger during navigation.
**Action:** Always ensure that custom interactive components (`div` acting as buttons or radios) implement both full ARIA attributes (`role`, `aria-checked`) and specific keyboard handlers (`e.key === 'Enter'`, `e.key === ' '` with `e.preventDefault()`) alongside `:focus-visible` outlines.
