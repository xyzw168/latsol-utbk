## 2026-05-20 - Custom Keyboard Accessibility for Interactive Elements
**Learning:** When making custom non-interactive elements (like 'div' elements for quiz options) interactive, relying solely on CSS hover/active states or click handlers is insufficient for accessibility. True keyboard accessibility requires setting 'role="radio"' (or similar), making them focusable with 'tabindex="0"', actively managing the 'aria-checked' state via JavaScript, and implementing a 'keydown' listener to handle 'Enter' and 'Space' keys (using 'e.preventDefault()' to prevent page scroll on Space).
**Action:** Always ensure that custom interactive elements created in JS loops explicitly manage focusability, ARIA states, and native keyboard interaction patterns (Enter/Space) in addition to mouse events, scoping DOM updates to the closest parent for performance.
