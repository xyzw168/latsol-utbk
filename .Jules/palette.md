## 2025-04-03 - Custom Radiogroup Keyboard Accessibility
**Learning:** When using custom interactive elements (like `div` elements representing quiz options) to function as a radiogroup, it is crucial to manually implement keyboard navigation (`Enter` and `Space` keys) and `aria-checked` states, as well as `tabindex`, because standard `div` elements do not possess these native accessibility features like standard `<input type="radio">` elements do. Without them, keyboard-only and screen reader users cannot properly interact with the quiz options.
**Action:** When creating custom selection components, always ensure they meet ARIA radiogroup/radio button requirements, including focus visibility, `role` definitions, dynamic `aria-checked` states, and keydown event listeners.
