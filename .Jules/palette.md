## 2025-05-25 - Implement Custom Keyboard Accessibility for Radio Options
**Learning:** Custom implementations of multiple-choice interactive components often lack native accessibility characteristics of form inputs (`<input type="radio">`). Just adding visual hover or focus states (`:focus-visible`) isn't enough; true a11y for non-semantic divs requires explicit ARIA attributes (`role="radiogroup"`, `role="radio"`, `aria-checked`, `aria-labelledby`), native focusability (`tabindex="0"`), and custom event listeners handling essential keyboard interactions ('Enter' and 'Space') to properly mirror semantic behavior for screen reader and keyboard users. Additionally, preventing default behavior on 'Space' keypress is critical to stop unintended page scrolling during option selection.
**Action:** When auditing or implementing interactive list options built with non-semantic elements (`div`, `span`), systematically verify the presence of `tabindex="0"`, `role="radio"`, `aria-checked` bindings, and comprehensive `keydown` handlers for Space and Enter that call the same selection logic as mouse clicks.
