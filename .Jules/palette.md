## 2026-05-11 - Accessible Custom Radio Buttons
**Learning:** When building custom interactive elements like quiz options (`.option` div elements), simply styling a selected state (`.selected`) is insufficient for keyboard and screen reader users. Screen readers need a context (`role="radiogroup"`) and individual element roles (`role="radio"`, `aria-checked`), while keyboard users need focusability (`tabindex="0"`) and specific event listeners (Enter/Space keys) since standard `div`s do not natively handle these interactions like `<input type="radio">`. Additionally, calling `e.preventDefault()` on the Space key is critical to prevent unintended page scrolling during keyboard navigation.
**Action:** When building interactive lists or options with non-semantic HTML (`div`, `span`), explicitly add ARIA roles, `tabindex`, and `keydown` event listeners mapped to the same behavior as the `onclick` handler to ensure full accessibility.
