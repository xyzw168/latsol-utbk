## 2024-05-18 - Keyboard Accessible Custom Radio Buttons
**Learning:** When using `div` elements as custom radio buttons (`role="radio"`), standard mouse events (`onclick`) are insufficient for accessibility. Users relying on keyboard navigation expect to be able to focus the element (`tabindex="0"`) and select it using `Enter` or `Space` keys. Furthermore, `aria-checked` must be manually managed to reflect the selected state.
**Action:** Implemented `keydown` event listeners on `.option` elements to listen for `Enter` and `Space` key presses, mapping them to the selection logic. Added `role="radiogroup"` to the container and `role="radio"`, `tabindex="0"`, and `aria-checked` to the individual options, ensuring the state is correctly toggled on selection. Scoped the option deselection query to the parent container (`box.querySelectorAll('.option')`) instead of `document.querySelectorAll('.option')` for better performance.
