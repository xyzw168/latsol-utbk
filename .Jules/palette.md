## 2024-05-24 - Interactive Quiz Options Keyboard Accessibility
**Learning:** Custom interactive elements representing radio buttons (like the quiz options) inherently lack keyboard accessibility and semantic meaning. Implementing 'role="radio"' within a 'role="radiogroup"', adding 'tabindex="0"', toggling 'aria-checked', and listening for Space/Enter keys ensures full keyboard and screen reader accessibility, aligning with WAI-ARIA authoring practices.
**Action:** Consistently apply the full set of ARIA attributes (`role="radio"`, `aria-checked`), focusability (`tabindex="0"`), and keyboard event handlers (Space/Enter) to custom interactive components that function as selection groups. Do not rely solely on `onclick`. Ensure `:focus-visible` styles are provided for visual focus indication during keyboard navigation.
