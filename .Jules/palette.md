## 2024-04-26 - Adding Keyboard Accessibility to Custom Radio Groups
**Learning:** When making custom quiz options (acting as radio buttons) keyboard accessible, simply adding `tabindex="0"` isn't enough. Screen readers need `role="radio"` and `aria-checked` attributes to understand the state, and the container needs `role="radiogroup"` with `aria-labelledby` to provide context. Furthermore, custom `onkeydown` handlers are required to trigger selection on 'Enter' and 'Space' keys, and it's critical to call `e.preventDefault()` on the 'Space' key to prevent the default page scrolling behavior.
**Action:** When implementing custom selection components, always provide the appropriate ARIA roles (e.g., `radiogroup`, `radio`), manage state explicitly (e.g., `aria-checked`), make elements focusable with `tabindex="0"`, and map keyboard interactions (Enter/Space) to the click handler while preventing default actions where necessary.
