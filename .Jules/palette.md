## 2026-04-26 - Custom Keyboard Accessibility for Radio Groups
**Learning:** When implementing custom radio groups using `div` elements, adding `:focus-visible` ensures clear visual feedback for keyboard users without affecting mouse users. Using `keydown` event listeners correctly capturing `Enter` and `Space` (with `e.preventDefault()` on `Space` to avoid scrolling) alongside `role="radiogroup"`, `role="radio"`, and `aria-checked` provides a solid screen reader and keyboard navigation experience. Also, scoping DOM queries to the parent container (`#options-box`) rather than the entire document minimizes traversal overhead.
**Action:** Always implement full ARIA attributes (`role`, `aria-checked`, `aria-labelledby`) and native-like keyboard interactions (`Space`/`Enter` with preventDefault) when building custom form controls like interactive option lists. Use `:focus-visible` for focus rings.
