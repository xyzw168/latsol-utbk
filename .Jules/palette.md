## 2025-02-28 - Custom Radio Group Keyboard Accessibility
**Learning:** Custom UI components built with nested `div`s lack native semantic meaning and keyboard navigation by default. When building interactive quiz options (e.g., `.option` elements), users navigating via keyboard cannot focus or select options using space/enter as they would with native `<input type="radio">` elements. Using `onclick` without matching keyboard event listeners excludes keyboard-only users. Furthermore, without proper ARIA roles (`role="radiogroup"`, `role="radio"`, and `aria-checked`), screen readers cannot announce the options' states or relationships. Calling `e.preventDefault()` on the spacebar keydown event is critical to stop the page from scrolling down when selecting an option.

**Action:** Whenever implementing custom interactive options that function like a radio group, apply `role="radiogroup"` on the parent container (with `aria-labelledby` referencing the question text) and `role="radio"` on each option. Make options focusable using `tabindex="0"`. Attach `keydown` event listeners to handle 'Enter' and ' ' (Space) keys, ensuring `e.preventDefault()` is used to prevent default scrolling behavior. Finally, dynamically synchronize the `aria-checked` states (`true` or `false`) alongside visual selected classes within the selection logic, and provide a clear `:focus-visible` outline for keyboard focus.
