## 2025-04-02 - Custom Radio Buttons Accessibility
**Learning:** When using `div` elements as custom radio buttons (`.option` elements inside a `.options` container), the lack of native keyboard accessibility prevents users from navigating and selecting options using a keyboard. Furthermore, even if `tabindex` and event listeners are added, the visual focus indicator might be missing, rendering keyboard navigation invisible to sighted users. Additionally, Playwright's `locator.focus()` might not reliably trigger CSS `:focus-visible` pseudo-class for validation scripts due to Chromium heuristics.
**Action:** Always implement full ARIA radiogroup/radio roles and state management (`aria-checked`) for custom `div`-based radio buttons. Add `keydown` event listeners for 'Enter' and 'Space' keys to trigger selection. Add clear `:focus-visible` CSS rules (e.g., `outline`) to ensure the focus state is visible. When verifying with Playwright, simulate keyboard presses (e.g., `page.keyboard.press('Tab')`) to accurately trigger and verify `:focus-visible` styles.
