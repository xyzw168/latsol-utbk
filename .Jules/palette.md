## 2024-03-20 - Adding Keyboard Accessibility to Custom Radio Groups
**Learning:** This app uses custom `div` elements for quiz options instead of native radio inputs. Without explicit ARIA roles and keyboard event listeners, these options are completely inaccessible to screen reader users and those navigating via keyboard. While standard W3C pattern uses a single tab-stop for the radiogroup and arrow keys to navigate, using `tabindex="0"` on all options with `Space`/`Enter` selection provides an acceptable lightweight fallback for basic keyboard accessibility.
**Action:** When working with custom UI components (like `.option` divs) acting as native form controls, always ensure they have the appropriate `role` (e.g., `radio`, `radiogroup`), `tabindex`, and handle `keydown` events (Space/Enter) to mimic native behavior. Use `:focus-visible` to provide clear visual feedback without affecting mouse users.
