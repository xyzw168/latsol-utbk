## 2026-05-31 - Keyboard Accessibility for Dynamic Custom Elements
**Learning:** When transforming `div` elements into custom radio buttons (`role="radio"`), standard mouse interaction (`onclick`) handles the logic perfectly, but keyboard users lack a way to activate them. Additionally, assigning `tabindex="0"` is necessary but insufficient alone; custom keydown listeners for `Enter` and `Space` are required to complete the interaction loop, and `e.preventDefault()` must be explicitly called for the `Space` key to prevent the browser from scrolling the page downwards instead of selecting the option.
**Action:** When implementing custom interactive elements (like `.option` in this quiz app), always pair `tabindex="0"` and `onclick` with a corresponding `keydown` listener that handles standard activation keys (`Enter`/`Space`) and manages default browser behaviors.
