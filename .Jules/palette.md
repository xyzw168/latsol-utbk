## 2026-05-23 - Interactive Quiz Option Accessibility
**Learning:** Custom interactive elements (like `<div>` acting as radio buttons) require explicit ARIA attributes (`role`, `aria-checked`), keyboard event listeners (`Space` and `Enter`), and container grouping (`radiogroup`, `aria-labelledby`) to be usable by screen readers and keyboard users. `tabindex="0"` makes them focusable, but custom `:focus-visible` styles are necessary to show focus state visually without creating unnecessary outlines for mouse users.
**Action:** When implementing custom interactive choices instead of native `<input type="radio">`, always establish a `radiogroup` container and assign `role="radio"`, `aria-checked`, `tabindex="0"`, and keydown handlers (Enter/Space) to the child elements. Provide visual focus using `:focus-visible`.
