## 2024-05-08 - Keyboard accessibility for custom radio options
**Learning:** Custom quiz options (`.option`) without native `<input type="radio">` lack keyboard focus out-of-the-box, breaking accessibility. Using `role="radio"`, `tabindex="0"`, `aria-checked`, and handling `Enter`/`Space` events is critical for screen readers and keyboard users. Spacebar default scroll should be prevented (`e.preventDefault()`). Also, querying `.option` inside the event handler is faster when scoped to the parent container (`box`) rather than the global document.
**Action:** Always implement full ARIA roles, `tabindex`, and keyboard event handlers (`keydown` for `Enter`/`Space`) when building custom interactive elements that act as options or buttons. Ensure focus visible styles are added.
