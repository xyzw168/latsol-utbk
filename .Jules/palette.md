## 2024-05-22 - Full Keyboard Support for Div-Based Interactive Elements
**Learning:** Replacing native HTML interactive elements (like `<input type="radio">` or `<button>`) with stylable `<div>` elements requires completely manual keyboard accessibility implementation. We learned that screen readers do not treat generic divs interactively unless explicit ARIA attributes (`role`, `aria-checked`), focusability (`tabindex`), and keyboard handlers (`keydown` for Enter/Space) are added. The Space key requires `e.preventDefault()` to stop the page from jumping/scrolling downwards.
**Action:** When creating custom components like the quiz options (`.option`), always ensure full native element semantics are manually restored if `<div>` wrappers are used instead of `<button>` or radio inputs.
