## 2026-05-21 - Custom Radio Group Accessibility
**Learning:** When building custom option select inputs, adding `tabindex="0"` isn't enough. Screen readers need `role="radiogroup"` on the container with an `aria-labelledby` referencing the question text. Individual options need `role="radio"` and their `aria-checked` attributes must be updated via JS whenever selection changes. Furthermore, keyboard-only users expect to select an option via the 'Space' or 'Enter' keys, and 'Space' events require `e.preventDefault()` to stop the page from scrolling down unexpectedly during selection.
**Action:** When implementing custom selection components in the future, always implement the full 'role="radio"' ARIA pattern, include a visual `:focus-visible` outline for Tab navigation, and attach `keydown` handlers for Space/Enter selection.
