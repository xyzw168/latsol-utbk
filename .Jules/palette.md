## 2024-05-18 - [Accessible Quiz Options]
**Learning:** A common accessibility anti-pattern was found where quiz options were rendered using `<div>` tags with `onclick` handlers, preventing keyboard navigation and screen reader support. Changing these elements to semantic `<button>` tags immediately added native keyboard support (tabbing and Enter/Spacebar activation). Furthermore, relying purely on visual cues like changing the border color or background to indicate selection is insufficient.
**Action:** Always use semantic HTML elements like `<button>` for interactive components. When building custom components, ensure keyboard support is added. Furthermore, always utilize ARIA attributes like `aria-pressed` to clearly communicate state changes (like selection) to assistive technologies, and combine this with visual focus indicators (`:focus-visible`) for keyboard users.
