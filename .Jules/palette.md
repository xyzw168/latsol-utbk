## 2026-05-15 - Custom Radio Button Accessibility
**Learning:** When using custom `.option` elements as alternatives to native `<input type="radio">` in the `#options-box` container, they inherently lack keyboard accessibility and screen reader semantics. Relying solely on `onclick` leaves out keyboard-only users, and focus management requires a distinct `:focus-visible` outline.
**Action:** Always assign `role="radiogroup"` to the parent container and `role="radio"`, `tabindex="0"`, and `aria-checked` to the individual interactive elements. Implement `keydown` event listeners for 'Enter' and 'Space' (using `e.preventDefault()` on Space to stop page scrolling) to trigger the selection logic synchronously with mouse events. Ensure CSS includes a high-contrast `:focus-visible` state.
