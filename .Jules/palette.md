## 2024-05-18 - Keyboard Accessible Custom Radio Groups
**Learning:** When using custom `div`-based radio options instead of native `<input type="radio">` elements, adding `tabindex="0"` is not enough. Screen readers and keyboard users also require explicit ARIA attributes (`role="radiogroup"`, `role="radio"`, `aria-checked`, `aria-labelledby`) and a `keydown` event listener to handle `Enter` and `Space` key activations. Furthermore, for the `Space` key, `e.preventDefault()` is crucial to prevent the page from unintentionally scrolling down when the user intends to select an option.
**Action:** When migrating or creating custom interactive components (like custom radio buttons), ensure full keyboard navigation parity by combining visual focus indicators (`:focus-visible`), appropriate ARIA roles and states, and manual keydown handlers that capture standard interaction keys.
