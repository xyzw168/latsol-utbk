## 2026-05-06 - Interactive Quiz Options Custom Keyboard Accessibility
**Learning:** Interactive options elements (rendered as divs within a list) lacked keyboard navigational accessibility out-of-the-box. When converting 'div' elements to act as interactive list-item options, adding native `<button>` or `<input type="radio">` is not always possible without restructuring styles. Implementing a custom keyboard accessibility pattern (roles, tabindex, event handlers) allows keyboard users to interact with options seamlessly while maintaining custom design requirements. Also learned that spacekey toggling via keyboard needs `preventDefault()` to stop the viewport from scrolling.
**Action:** When working with custom non-native interactive items, always ensure ARIA compliance ('role="radiogroup"' for containers and 'role="radio"' for elements), assign proper attributes (`tabindex="0"`, `aria-checked`), and add custom keyboard listeners that map to selection functionality (both "Enter" and "Space" key), remembering to prevent default scrolling for the space key. Provide a visual outline using `:focus-visible` pseudo-class rather than generic `:focus` to only affect keyboard users.
