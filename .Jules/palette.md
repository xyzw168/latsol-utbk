## 2026-04-02 - Custom Keyboard Accessibility for Interactive Divs
**Learning:** Interactive UI components built with `div` instead of native form elements lack built-in keyboard navigation and screen-reader context. When using `div` as options (radios), users relying on keyboard navigation or screen readers cannot select options or understand their state. Using `document.querySelectorAll` inside frequently triggered events (like toggling selection) causes unnecessary layout traversal, impacting responsiveness.
**Action:** Always add semantic roles (`role="radiogroup"`, `role="radio"`), keyboard navigability (`tabindex="0"`), state indicators (`aria-checked`), and visual feedback (`:focus-visible`) to custom elements. Implement `keydown` event listeners for 'Enter' and 'Space' to mimic native interaction. Scope DOM traversal (e.g. `querySelectorAll`) to the immediate parent container rather than `document` to maintain performance best practices.
