## 2026-03-31 - Keyboard Accessibility for Custom Radio Groups
**Learning:** When creating custom radio buttons using non-native elements like `<div>`, it's critical to implement keyboard events manually in addition to adding the correct ARIA attributes (`role="radio"`, `aria-checked`, `tabindex="0"`). Without explicit `keydown` event listeners for `Enter` and `Space`, screen reader users and keyboard-only users will not be able to interact with the options, breaking accessibility completely. Furthermore, adding `:focus-visible` styles is essential so that visual focus indicators are shown to keyboard users without negatively impacting the mouse user experience.
**Action:** Always pair `role="radio"` with a corresponding `keydown` listener that delegates `Enter` or `Space` presses to the selection handler (e.g., `div.click()`) and ensure a clear `.element:focus-visible` state exists in CSS when building custom interactive UI components.
