## 2025-04-13 - Custom Radio Group Accessibility
**Learning:** When implementing a custom radiogroup utilizing native elements like `div` without inherent radio semantic behaviors, explicitly associating them via `role="radiogroup"`, `role="radio"`, and `aria-checked` isn't enough for keyboard accessibility. When mapping keydown events for 'Space', it explicitly triggers page scrolling (which is default browser behavior) unless explicitly prevented via `e.preventDefault()`, which disrupts keyboard selection flows.
**Action:** Always include `e.preventDefault()` within Space/Enter keydown event listeners on custom focusable elements acting as inputs, and explicitly scope focus selections inside components via standard container references rather than full document queries to prevent unintended element bleeding.
