## 2024-05-24 - Interactive quiz options require custom ARIA and keyboard bindings
**Learning:** When using custom elements like `div` to represent interactive list items (such as multiple-choice options), it's crucial to explicitly provide ARIA roles (`role="radio"`, `role="radiogroup"`), keyboard focusability (`tabindex="0"`), accessible states (`aria-checked`), and manual keyboard event handling (for `Enter` and `Space` keys) since these elements do not inherently receive them unlike native `<button>` or `<input type="radio">` tags. In addition, providing a clear visual focus indicator (e.g. `:focus-visible`) helps keyboard users navigate without disrupting mouse users.
**Action:** Next time I work with custom interactive lists or options in a custom component, I will proactively map the necessary ARIA attributes, semantic roles, keyboard event handlers (`onkeydown`), and CSS focus outlines to ensure they function properly for all users.
