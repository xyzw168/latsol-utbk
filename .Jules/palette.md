## 2024-05-15 - Interactive Quiz Options Keyboard Accessibility
**Learning:** Custom list elements functioning as quiz options require proper ARIA attributes (`role="radiogroup"`, `role="radio"`, `aria-checked`) and specific keyboard event listeners (`keydown` for Enter and Space) to ensure they are accessible via keyboard navigation. Simulated focus using Javascript does not trigger `:focus-visible` pseudo-class styles in headless Chromium. Simulated keyboard navigation (`Tab`) is required to trigger `:focus-visible`.
**Action:** When implementing custom interactive lists or options without native `<input type="radio">` or `<button>` elements, always manually add standard ARIA roles, states, keyboard navigation listeners, and `focus-visible` outlines. In Playwright scripts, simulate physical tab keys rather than programmatic `.focus()` calls.
